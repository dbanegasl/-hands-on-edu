## Resumen

La versión 0.1.0 representa el **primer hito funcional** de HandsOnEdu como plataforma educativa para la Universidad Nacional de Educación (UNAE) de Ecuador. Este lanzamiento establece la base tecnológica completa del proyecto: cuatro módulos educativos operativos, una infraestructura de backend en tiempo real con MediaPipe, y la arquitectura Docker que permite despliegue reproducible en cualquier entorno institucional.

Esta versión es una **prueba de concepto validada en aula**: demuestra que la detección de gestos de mano con cámara web estándar puede ser una herramienta pedagógica real para evaluaciones, aprendizaje de lenguaje de señas, control de asistencia y expresión artística aumentada. Sienta las bases sobre las cuales se construirán las funcionalidades de seguimiento y análisis del aprendizaje en versiones futuras.

## Módulos incluidos

### 🎓 GestiEdu — Evaluaciones por Gestos

Sistema de evaluación gamificada donde los estudiantes responden preguntas usando gestos de mano en lugar de clics o teclado. Transforma la evaluación formativa en una actividad kinestésica y participativa.

- **Mecánica:** 10 preguntas de opción múltiple (A, B, C, D) mapeadas a gestos (pulgar arriba, señalando, paz, puño).
- **Gamificación:** sistema de puntos, tiempo límite por pregunta, pantalla de resultados con calificación 0–10.
- **Retroalimentación:** visual inmediata (color verde/rojo) con indicación del gesto correcto.
- **Uso pedagógico:** ideal para repasos de contenido, diagnósticos de entrada, actividades de calentamiento.

### 🤟 MotivaSign — Lenguaje de Señas

Módulo de aprendizaje progresivo de lenguaje de señas Ecuatoriano. Diseñado para concienciar a la comunidad UNAE sobre la comunicación inclusiva y acercar la lengua de señas a estudiantes de educación.

- **Progresión:** sistema de niveles con dificultad creciente.
- **Reconocimiento en tiempo real:** el sistema detecta si el signo realizado coincide con el objetivo.
- **Modo libre:** práctica sin restricciones de tiempo ni puntuación.
- **Impacto:** herramienta de sensibilización sobre discapacidad auditiva para futuros docentes.

### 👁️ AttendEye — Asistencia Gestual

Sistema de registro de asistencia que reemplaza el proceso manual de llamada de lista. Los estudiantes registran su presencia haciendo un gesto frente a la cámara.

- **Check-in:** gesto de mano abierta frente a la cámara = registro de presencia.
- **Lista en tiempo real:** el docente ve qué estudiantes han registrado asistencia.
- **Reporte de sesión:** resumen de presentes/ausentes al finalizar la clase.
- **Ahorro de tiempo:** elimina los 5–10 minutos de llamada de lista al inicio de clase.

### 🎨 VirtualPainter — Dibujo en Realidad Aumentada

Herramienta de expresión artística que usa el dedo índice como pincel virtual sobre el feed de la cámara web. Especialmente diseñado para actividades creativas con niños.

- **Dibujo libre:** trazado en tiempo real con punta del dedo índice.
- **Paleta de colores:** selección de color por zona de la pantalla (sin menús, solo gestos).
- **Eraser:** gesto específico para borrar trazos.
- **Limpiar lienzo:** botón para empezar desde cero.
- **Uso pedagógico:** actividades de arte digital, ilustración de conceptos, expresión libre.

## Testing Lab

El Testing Lab es un entorno de diagnóstico y exploración para docentes y desarrolladores. Permite verificar que el hardware y la conexión funcionan correctamente antes de usar los módulos en clase, y explorar las capacidades de MediaPipe en tiempo real.

**6 modos disponibles:**

| Modo | Descripción |
|------|-------------|
| Detección de gestos | Muestra el gesto detectado en tiempo real con su etiqueta |
| Conteo de dedos | Número de dedos levantados en cada mano |
| Tracking de punta | Coordenadas de la punta del dedo índice en píxeles |
| Visualización de landmarks | Los 21 puntos del esqueleto de la mano dibujados sobre el feed |
| Modo espejo | Feed espejado para verificar coordinación intuitiva |
| Debug de landmarks | Dump en pantalla de todos los valores numéricos de landmarks |

## Infraestructura

### Stack tecnológico

| Componente | Tecnología |
|-----------|-----------|
| Backend | FastAPI (Python 3.11) |
| Visión computacional | MediaPipe Hand Landmarker |
| Comunicación | WebSocket (`/ws/analyze`) |
| Frontend | Vanilla JavaScript (ES2022 modules) |
| Contenedor | Docker + docker-compose |
| Puerto de exposición | **9876** |

### Arquitectura de procesamiento en tiempo real

```
[Navegador]                    [Servidor Docker :9876]
    │                                    │
    │  WebSocket /ws/analyze             │
    ├──── Frame JPEG (640×480) ─────────►│
    │                                    │  MediaPipe
    │                                    │  HandTracker
    │◄─── JSON {gesture, fingers,  ─────┤  detect_gesture()
    │           landmarks, coords}       │
    │                                    │
    │  Renderizar overlay en canvas      │
```

## Limitaciones conocidas (deuda técnica)

Esta versión es una prueba de concepto funcional, no un producto de producción. Las siguientes limitaciones son conocidas y están documentadas con planes de resolución:

| Limitación | Impacto | Ver |
|-----------|---------|-----|
| **Sin tests automatizados** (0% cobertura) | Refactors riesgosos, sin CI/CD | [TD-001](../technical-debt/TECHNICAL-DEBT.md#td-001), [REQ-001](../requirements/REQ-001-unit-tests.md) |
| **Sin autenticación** (plataforma pública) | Inaceptable en producción institucional | [TD-003](../technical-debt/TECHNICAL-DEBT.md#td-003), [REQ-008](../requirements/REQ-008-basic-auth.md) |
| **Sin persistencia** (datos en memoria) | Resultados se pierden al reiniciar | [TD-004](../technical-debt/TECHNICAL-DEBT.md#td-004), [REQ-005](../requirements/REQ-005-sqlite-persistence.md) |
| **Preguntas hardcodeadas** en JS | Docente no puede personalizar | [TD-002](../technical-debt/TECHNICAL-DEBT.md#td-002), [REQ-006](../requirements/REQ-006-question-editor-gestiedu.md) |
| **MoodleClient es stub** | Integración LMS no funcional | [TD-005](../technical-debt/TECHNICAL-DEBT.md#td-005), [REQ-009](../requirements/REQ-009-moodle-integration.md) |
| **Sin rate limiting en WebSocket** | Vulnerable a saturación | [TD-008](../technical-debt/TECHNICAL-DEBT.md#td-008) |
